*2) INFORMES QUE ENVIAMOS EL 21 FEB 2026 — Resumen y qué necesitamos*

*2.1 BUG CRÍTICO — Lo que ve el usuario (stream=true)*
En el chat del Copilot (modo streaming) el usuario recibe siempre: "Se produjo un error de comunicación con el servidor."
Respuesta real: HTTP 200 con body `NO_PROVIDERS_AVAILABLE` (trace_id 0088d0be). Con stream=false sí intenta anthropic/deepseek-chat (y falla por configuración). *Pregunta:* ¿Por qué con stream=true no hay ningún proveedor disponible y con stream=false sí?

*2.2 AUTH_ERROR (30/80 preguntas) y PROVIDER_ERROR 400 (27/80)*
Causa confirmada: `provider=anthropic` + `model=deepseek-chat` (combinación inválida). Upstream 401/400. *Acción en api-ia/API2:* Cambiar provider a deepseek con key DeepSeek, o model a claude-3-5-haiku (o similar) con key Anthropic correcta.

*2.3 Respuestas sin contexto de usuario (22/80 HTTP 200 pero ilógicas)*
El modelo responde "necesito saber el nombre del evento" en preguntas como "¿Cuántos invitados tengo?" o "Agrega a Jose Garcia como invitado", aunque el usuario tiene sesión y evento activo. *Preguntas:* ¿api-ia necesita recibir el JWT de Firebase del usuario para inyectar contexto del evento? ¿O es API2 quien debe enviar ese contexto?

*2.4 GET /webapi/models/anthropic → array vacío []*
*Pregunta:* ¿Es esperado o hay un problema de registro de modelos?

*2.5 POST /api/ai/improve → HTTP 500*
"Tarea no encontrada: backend.workers.granular_ai.improve_text". *Pregunta:* ¿Está desplegado el worker de Celery para improve_text?
