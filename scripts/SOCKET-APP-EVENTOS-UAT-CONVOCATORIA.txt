• *Front/deploy:* la build de la app debe llevar `NEXT_PUBLIC_SOCKET_URL` (o la base acordada) **exactamente** apuntando a esa URL (prioridad sobre `NEXT_PUBLIC_BASE_API_BODAS`; ver `apps/appEventos/api.js` → `resolveSocketBaseUrl`).

2. PRERREQUISITOS (antes de convocar)
--------------------------------------------------------------------------------
  [ ] Host socket desplegado y estable (api-ia backend según §12 especificación).
  [ ] DNS público resuelve al host correcto; certificado válido (HTTPS).
  [ ] `…/socket.io/?EIO=4&transport=polling` devuelve cuerpo típico engine.io (no HTML error).
  [ ] Variable `NEXT_PUBLIC_SOCKET_URL` aplicada en el **mismo** entorno donde abrirán la app
      (Vercel / env del build; requiere redeploy si se añade o cambia).
  [ ] Testers pueden iniciar sesión en la app y abrir el mismo evento (IDs conocidos).
